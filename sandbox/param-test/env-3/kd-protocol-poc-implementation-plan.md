# KD Protocol Benchmarking PoC — Local Setup Guide

> **Scope:** Classical, hybrid, and pure post-quantum **key distribution (KD) protocol** benchmarking using OQS-Nginx and OQS-Locust on Windows/Docker Desktop (WSL 2 backend).
>
> **Signature scheme:** ECDSA P-256 (classical) for all certificates — only the *key exchange* (KEM group) varies across test runs.
>
> **TLS version:** TLS 1.3 only.

## 2. Cryptographic Credentials Generation

Use the `openquantumsafe/oqs-ossl3` Docker image — it ships with OpenSSL 3 + OQS provider pre-built, with no host installation needed.

> **Why ECDSA P-256 for signing?**
> This PoC isolates the **key distribution (KEM) layer** as the variable under test. Using a classical ECDSA P-256 signature scheme for the CA and server certificates keeps the certificate stack well-understood and broadly compatible, so any performance difference observed across test runs is attributable purely to the KEM group negotiated — not the signature algorithm.
>
> The `openquantumsafe/oqs-ossl3` image includes a full OQS OpenSSL build, but standard `prime256v1` (P-256) key generation works identically to native OpenSSL.

> ⚠️ Run these commands from the **pqc-poc/** root directory. The `${PWD}/certs` bind-mount writes files into your local `certs/` folder. On Windows PowerShell, replace `$(pwd)` with `${PWD}`.

**`generate-certs.sh`**
```bash
# ── Step 2a: Generate the CA private key using ECDSA P-256 ───────────────
docker run --rm \
  -v "${PWD}/certs:/certs" \
  openquantumsafe/oqs-ossl3 \
  openssl genpkey \
    -algorithm EC \
    -pkeyopt ec_paramgen_curve:P-256 \
    -out /certs/ca.key

# ── Step 2b: Self-sign the CA certificate (10-year validity for PoC) ─────
docker run --rm \
  -v "${PWD}/certs:/certs" \
  openquantumsafe/oqs-ossl3 \
  openssl req -new -x509 \
    -days 3650 \
    -key  /certs/ca.key \
    -out  /certs/ca.crt \
    -subj "/CN=KD-PoC-CA/O=KD-Lab/C=US"

# ── Step 2c: Generate server private key (same classical algorithm) ───────
docker run --rm \
  -v "${PWD}/certs:/certs" \
  openquantumsafe/oqs-ossl3 \
  openssl genpkey \
    -algorithm EC \
    -pkeyopt ec_paramgen_curve:P-256 \
    -out /certs/server.key

# ── Step 2d: Create server CSR (CN must match the Docker service name) ───
docker run --rm \
  -v "${PWD}/certs:/certs" \
  openquantumsafe/oqs-ossl3 \
  openssl req -new \
    -key  /certs/server.key \
    -out  /certs/server.csr \
    -subj "/CN=oqs-nginx/O=KD-Lab/C=US"

# ── Step 2e: Sign the server cert with the CA ────────────────────────────
docker run --rm \
  -v "${PWD}/certs:/certs" \
  openquantumsafe/oqs-ossl3 \
  openssl x509 -req \
    -days    365 \
    -in      /certs/server.csr \
    -CA      /certs/ca.crt \
    -CAkey   /certs/ca.key \
    -CAcreateserial \
    -out     /certs/server.crt

# ── Step 2f: Verify the chain looks correct ───────────────────────────────
docker run --rm \
  -v "${PWD}/certs:/certs" \
  openquantumsafe/oqs-ossl3 \
  openssl verify \
    -CAfile /certs/ca.crt \
    /certs/server.crt
# Expected output: /certs/server.crt: OK
```

> 💡 **Alternative classical signature algorithms** supported by `openquantumsafe/oqs-ossl3` (all suitable for this PoC): `prime256v1` (P-256, recommended), `secp384r1` (P-384), `secp521r1` (P-521), `rsa:2048`, `rsa:4096`. P-256 offers the best balance of handshake speed and key size for benchmarking purposes.

## 3. Nginx Configuration

The `ssl_ecdh_curves` directive controls which KEM groups are advertised during the TLS 1.3 key exchange. **Order matters** — the server prefers left-to-right. The client (Locust) will choose the first group it also supports.

### KEM Groups Under Test

| Group Name       | Type         | NIST Security Level     | Notes                                    |
|------------------|--------------|-------------------------|------------------------------------------|
| `X25519_MLKEM768`| Hybrid       | Level 3 (~AES-192)      | Preferred — classical + PQ combined KEM  |
| `P384_MLKEM1024` | Hybrid       | Level 5 (~AES-256)      | Higher security hybrid option            |
| `mlkem768`       | Pure PQ      | Level 3                 | Pure post-quantum, no classical component|
| `X25519`         | Classical    | ~Level 3 (classical)    | Fallback for non-PQ clients              |
| `P-256`          | Classical    | ~Level 1 (classical)    | Broad compatibility fallback             |

> ℹ️ The certificate uses ECDSA P-256 (classical) for authentication. The KEM groups above govern the *key exchange* only — the two are independent. This is the intended configuration: a trusted classical cert signs the session, while the KEM group is the variable being benchmarked.

**`nginx/nginx.conf`**
```nginx
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
#  OQS-Nginx KD Protocol Benchmarking — TLS 1.3 Configuration
#  Image: openquantumsafe/nginx:latest
#  Signature: ECDSA P-256 (classical)  |  KEM: variable (see ssl_ecdh_curves)
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

worker_processes  1;    # Match CPU limit in docker-compose
error_log         /var/log/nginx/error.log info;
pid               /var/run/nginx.pid;

events {
    worker_connections 1024;
    use                epoll;
    multi_accept       on;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    # ── Logging: include $ssl_curve for handshake group visibility ──────
    log_format kd_fmt '$remote_addr - [$time_local] "$request" '
                      '$status $body_bytes_sent '
                      'ssl_protocol=$ssl_protocol '
                      'ssl_cipher=$ssl_cipher '
                      'ssl_curve=$ssl_curve '        # <-- KEY FIELD
                      'rt=$request_time';

    access_log /var/log/nginx/access.log kd_fmt;

    sendfile    on;
    tcp_nopush  on;
    keepalive_timeout  65;

    # ── HTTPS Server Block ──────────────────────────────────────────────
    server {
        listen       443 ssl;
        server_name  oqs-nginx;

        # Certificate chain (ECDSA P-256 signed, generated in Step 2)
        ssl_certificate      /certs/server.crt;
        ssl_certificate_key  /certs/server.key;

        # ── TLS Protocol: strict TLS 1.3 only ───────────────────────────
        ssl_protocols  TLSv1.3;

        # ── KEM Groups advertised to clients ────────────────────────────
        #  Order of preference (left = most preferred):
        #    1. X25519_MLKEM768    → HYBRID  (classical + PQ)     ★ default
        #    2. P384_MLKEM1024     → HYBRID  (classical + PQ, higher sec)
        #    3. mlkem768           → PURE PQ (no classical component)
        #    4. X25519             → CLASSICAL
        #    5. P-256              → CLASSICAL broad-compat
        ssl_ecdh_curves  X25519_MLKEM768:P384_MLKEM1024:mlkem768:X25519:P-256;

        # ── TLS 1.3 ciphersuites ────────────────────────────────────────
        ssl_ciphers  TLS_AES_256_GCM_SHA384:TLS_AES_128_GCM_SHA256:TLS_CHACHA20_POLY1305_SHA256;

        # ── Session settings ────────────────────────────────────────────
        ssl_session_cache    shared:SSL:10m;
        ssl_session_timeout  10m;
        ssl_session_tickets  off;  # Disable for consistent full handshakes

        # ── HSTS (informational for PoC) ────────────────────────────────
        add_header Strict-Transport-Security "max-age=300" always;
        add_header X-KD-Group               "$ssl_curve" always;

        # ── Static content ──────────────────────────────────────────────
        root   /usr/share/nginx/html;
        index  index.html;

        location / {
            try_files $uri $uri/ =404;
        }

        # ── Health check endpoint ───────────────────────────────────────
        location /health {
            return 200 "OK ssl_curve=$ssl_curve\n";
            add_header Content-Type text/plain;
        }
    }
}
```

## 4. Locust Traffic Generator

Complete `locustfile.py` for the `openquantumsafe/locust` image. Configures OpenSSL to use OQS groups and exercises classical, hybrid, and pure-PQ key distribution handshakes.

> 💡 **How OQS groups are forced:** The `openquantumsafe/locust` image ships with Python bound to an OQS-enabled OpenSSL build. The `ssl.SSLContext` created in the script sets `set_groups()` (available as `set_ciphers` for groups in older bindings) or via the `OPENSSL_GROUPS` environment variable. We use an `env` variable approach in docker-compose as the most reliable method across image versions, and set it programmatically in Python as a belt-and-suspenders measure.

Configuration knobs:

- **X (concurrent users):** Set via Locust UI or `--users` CLI flag. Default: 10 users.
- **Y (total iterations):** Controlled by `NUM_REQUESTS` env var.
- **Z (delay between requests):** Controlled by `WAIT_MIN` / `WAIT_MAX` env vars.

**`locust/locustfile.py`**
```python
"""
KD Protocol Benchmarking PoC — Locust load test script.
Image: openquantumsafe/locust

Signature scheme: ECDSA P-256 (classical) — fixed for all runs.
Variable under test: KEM group (key distribution protocol).

Configuration knobs (set via env vars or edit defaults below):
  WAIT_MIN      Z_low  — minimum seconds between requests  (default: 1.0)
  WAIT_MAX      Z_high — maximum seconds between requests  (default: 3.0)
  NUM_REQUESTS  Y      — stop after N requests per user   (default: 50)
  OQS_KEM_GROUP        — TLS KEM group to negotiate       (default: X25519_MLKEM768)
  TARGET_HOST          — HTTPS base URL of the server
"""

import os
import ssl
import time
import logging
import urllib3
from locust import HttpUser, task, between, events
from locust.exception import StopUser

# ── Configuration ────────────────────────────────────────────────────────
WAIT_MIN      = float(os.getenv("WAIT_MIN",      "1.0"))
WAIT_MAX      = float(os.getenv("WAIT_MAX",      "3.0"))
NUM_REQUESTS  = int(os.getenv("NUM_REQUESTS",  "50"))
KEM_GROUP     = os.getenv("OQS_KEM_GROUP",   "X25519_MLKEM768")
TARGET_HOST   = os.getenv("TARGET_HOST",     "https://oqs-nginx:443")

# Belt-and-suspenders: also set the env var so the OQS OpenSSL picks it up
os.environ["OPENSSL_GROUPS"] = KEM_GROUP

logging.basicConfig(level=logging.INFO)
log = logging.getLogger("kd-locust")
log.info(f"KEM group requested: {KEM_GROUP}")
log.info(f"Requests per user:   {NUM_REQUESTS}")
log.info(f"Wait range (s):      [{WAIT_MIN}, {WAIT_MAX}]")

# Disable urllib3 SSL warnings — we use a self-signed CA intentionally
urllib3.disable_warnings(urllib3.exceptions.InsecureRequestWarning)


def build_oqs_ssl_context(kem_group: str) -> ssl.SSLContext:
    """
    Build an SSLContext that:
    - Enforces TLS 1.3 only
    - Requests the specified KEM group for key exchange
    - Skips server cert verification (self-signed PoC CA)
    """
    ctx = ssl.SSLContext(ssl.PROTOCOL_TLS_CLIENT)
    ctx.minimum_version = ssl.TLSVersion.TLSv1_3
    ctx.maximum_version = ssl.TLSVersion.TLSv1_3
    ctx.check_hostname  = False
    ctx.verify_mode     = ssl.CERT_NONE  # PoC only — use real CA in staging

    # set_groups() is the correct Python 3.10+ API
    # Falls back to set_ciphers for named group on older builds
    try:
        ctx.set_groups(kem_group)
        log.info(f"SSLContext groups set to: {kem_group}")
    except AttributeError:
        # Older OQS Python binding — use OPENSSL_GROUPS env var (set above)
        log.warning("set_groups() not available; relying on OPENSSL_GROUPS env var")

    return ctx


# ── Locust Event Hooks ───────────────────────────────────────────────────
@events.init.add_listener
def on_locust_init(environment, **kwargs):
    log.info(f"Locust initialised — target: {TARGET_HOST}, group: {KEM_GROUP}")


@events.request.add_listener
def on_request(request_type, name, response_time, response_length,
               exception, context, **kwargs):
    """Log KD-relevant details for every request."""
    if exception:
        log.error(f"[FAIL] {request_type} {name} — {exception}")
    else:
        # response_time is in milliseconds
        log.debug(f"[OK]   {request_type} {name} — {response_time:.1f}ms")


# ── User Class ───────────────────────────────────────────────────────────
class KDUser(HttpUser):
    """
    Simulates a user making HTTPS requests to benchmark key distribution protocols.

    X (concurrent users) = --users flag to Locust CLI / Locust UI
    Y (total requests)   = NUM_REQUESTS env var (stops after this many)
    Z (delay between)    = random uniform in [WAIT_MIN, WAIT_MAX] seconds
    """
    host         = TARGET_HOST
    wait_time    = between(WAIT_MIN, WAIT_MAX)   # Z seconds
    abstract     = False

    def on_start(self):
        """Called once per simulated user on spawn."""
        self.request_count = 0
        # Patch the session's SSL adapter to use our OQS context
        oqs_ctx = build_oqs_ssl_context(KEM_GROUP)
        adapter = _OQSHTTPSAdapter(ssl_context=oqs_ctx)
        self.client.mount("https://", adapter)
        log.info(f"User spawned — OQS SSL adapter mounted (group={KEM_GROUP})")

    @task(10)
    def get_homepage(self):
        """
        Main benchmark task: GET /
        Weight 10 — highest frequency task.
        """
        self._do_request("/", name="/ [homepage]")

    @task(3)
    def get_health(self):
        """
        Health probe: GET /health — returns ssl_curve in body.
        Weight 3 — less frequent; useful for spot-checking the negotiated group.
        """
        with self.client.get(
            "/health",
            name="/health [probe]",
            verify=False,
            catch_response=True
        ) as resp:
            if resp.status_code == 200:
                log.info(f"Health body: {resp.text.strip()}")
                if KEM_GROUP.lower() not in resp.text.lower():
                    # Server echoes ssl_curve; warn if it doesn't match what we asked for
                    log.warning(
                        f"Negotiated group mismatch! "
                        f"Asked: {KEM_GROUP}, Server reported: {resp.text.strip()}"
                    )
                resp.success()
            else:
                resp.failure(f"Unexpected status: {resp.status_code}")

        self.request_count += 1
        self._check_stop()

    def _do_request(self, path: str, name: str = None):
        with self.client.get(
            path,
            name=name or path,
            verify=False,
            catch_response=True
        ) as resp:
            if resp.status_code == 200:
                resp.success()
            else:
                resp.failure(f"HTTP {resp.status_code}")

        self.request_count += 1
        self._check_stop()

    def _check_stop(self):
        """Stop this user after Y total requests."""
        if self.request_count >= NUM_REQUESTS:
            log.info(f"User reached {NUM_REQUESTS} requests — stopping.")
            raise StopUser()


# ── Custom HTTPS Adapter ─────────────────────────────────────────────────
try:
    from requests.adapters import HTTPAdapter
    from urllib3.util.ssl_ import create_urllib3_context

    class _OQSHTTPSAdapter(HTTPAdapter):
        """Injects an OQS-aware SSLContext into the requests session."""
        def __init__(self, ssl_context=None, **kwargs):
            self._ssl_ctx = ssl_context
            super().__init__(**kwargs)

        def init_poolmanager(self, *args, **kwargs):
            if self._ssl_ctx:
                kwargs["ssl_context"] = self._ssl_ctx
            super().init_poolmanager(*args, **kwargs)

        def proxy_manager_for(self, proxy, **proxy_kwargs):
            if self._ssl_ctx:
                proxy_kwargs["ssl_context"] = self._ssl_ctx
            return super().proxy_manager_for(proxy, **proxy_kwargs)

except ImportError as exc:
    raise RuntimeError(f"requests/urllib3 not available: {exc}")
```

## 5. Docker Compose Orchestration

Single file that brings up the entire PoC stack: OQS-Nginx + OQS-Locust on a shared bridge network, with CPU/memory limits.

- **Resource limits:** Each container is pinned to 1 CPU core and 512 MB RAM via `deploy.resources.limits`. Adjust `cpus` downward on low-spec machines.
- **Health check dependency:** Locust uses `depends_on: condition: service_healthy` so it won't fire until Nginx passes its health check — preventing false connection errors.

**`docker-compose.yml`**
```yaml
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
#  KD Protocol Benchmarking PoC — Docker Compose
#  Signature: ECDSA P-256 (classical) | KEM groups: classical / hybrid / pure-PQ
#  Usage: docker compose up --build
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

version: "3.9"

networks:
  pqc-net:
    driver: bridge
    name:   pqc-net

services:

  # ── OQS-Nginx: TLS 1.3 server with classical/hybrid/pure-PQ KEM groups ─
  oqs-nginx:
    image: openquantumsafe/nginx:latest
    container_name: oqs-nginx
    hostname:       oqs-nginx        # Must match CN in server.crt
    networks:
      - pqc-net
    ports:
      - "443:443"                    # Expose HTTPS to host for manual curl tests
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/index.html:/usr/share/nginx/html/index.html:ro
      - ./certs:/certs:ro
    environment:
      # Advertise all three protocol classes; server prefers left-to-right
      - OPENSSL_GROUPS=X25519_MLKEM768:P384_MLKEM1024:mlkem768:X25519:P-256
    deploy:
      resources:
        limits:
          cpus:   "1.0"             # Pin to 1 CPU core
          memory: 256M
        reservations:
          cpus:   "0.25"
          memory: 64M
    healthcheck:
      # Use OQS openssl inside the container to verify TLS 1.3 + KEM group
      test: ["CMD-SHELL", "openssl s_client -connect localhost:443 -tls1_3 -groups X25519_MLKEM768 -CAfile /certs/ca.crt </dev/null 2>&1 | grep -q 'Verify return code: 0'"]
      interval:     10s
      timeout:      5s
      retries:      5
      start_period: 8s
    restart: on-failure

  # ── OQS-Locust: KD-protocol-aware load tester ────────────────────────
  oqs-locust:
    image: openquantumsafe/locust:latest
    container_name: oqs-locust
    networks:
      - pqc-net
    ports:
      - "8089:8089"                  # Locust Web UI
    volumes:
      - ./locust:/mnt/locust:ro
      - ./certs:/certs:ro            # Available for optional CA pinning
    environment:
      # ── Locust core settings ─────────────────────────────────────────
      - LOCUST_LOCUSTFILE=/mnt/locust/locustfile.py
      - LOCUST_HOST=https://oqs-nginx:443
      # ── KEM group under test — change this to switch protocol class ──
      - OQS_KEM_GROUP=X25519_MLKEM768   # hybrid  | mlkem768 = pure-PQ | X25519 = classical
      - OPENSSL_GROUPS=X25519_MLKEM768  # Belt-and-suspenders: env-level OQS hint
      # ── Traffic shape knobs (X, Y, Z) ────────────────────────────────
      - WAIT_MIN=1.0                    # Z low  — min seconds between requests
      - WAIT_MAX=3.0                    # Z high — max seconds between requests
      - NUM_REQUESTS=50                 # Y      — requests per user before stop
    command: >
      locust
        --locustfile /mnt/locust/locustfile.py
        --host https://oqs-nginx:443
        --web-port 8089
      # Remove the next line to use the Web UI instead of headless mode:
      # --headless --users 10 --spawn-rate 2 --run-time 120s
    depends_on:
      oqs-nginx:
        condition: service_healthy
    deploy:
      resources:
        limits:
          cpus:   "1.0"
          memory: 512M
        reservations:
          cpus:   "0.25"
          memory: 128M
    restart: on-failure
```

## 6. Execution & Verification

Chronological commands to bring the stack up, run tests against each key distribution protocol class, and confirm that the correct KEM group was negotiated.

```
Host (Windows)  →  oqs-locust (:8089)  ⇌ TLS 1.3 / KEM group  ⇌  oqs-nginx (:443)
Docker Desktop / WSL2          Both containers on shared pqc-net bridge network
```

### Step 1 — Pull the OQS images *(only needed once)*

```bash
docker pull openquantumsafe/oqs-ossl3:latest
docker pull openquantumsafe/nginx:latest
docker pull openquantumsafe/locust:latest
```

### Step 2 — Generate certificates

Run the five `docker run` commands from Section 2, then verify:

ls -lh certs/
# Expect: ca.key  ca.crt  server.key  server.csr  server.crt
```

docker compose up -d

# Watch startup logs

```

Wait until `oqs-nginx` reaches **healthy** status. Locust will start automatically after.

### Step 4 — Verify Nginx health from the host

```bash
# Option A: Skip cert verification (quick PoC check)
curl -kv https://localhost/health 2>&1 | grep -E "ssl_curve|TLSv|issuer|subject"

# Option B: Verify against our CA cert
curl --cacert certs/ca.crt https://localhost/health
```

The `/health` endpoint echoes the negotiated group: `OK ssl_curve=X25519_MLKEM768`

### Step 5 — Confirm the handshake with `openssl s_client`

```bash
docker run --rm --network pqc-net \
  -v "${PWD}/certs:/certs" \
  openquantumsafe/oqs-ossl3 \
  openssl s_client \
    -connect   oqs-nginx:443 \
    -tls1_3 \
    -groups    X25519_MLKEM768 \
    -CAfile    /certs/ca.crt \
    -brief </dev/null 2>&1 | head -30
```

Look for these lines — they confirm the expected handshake:

```
Protocol  : TLSv1.3
Server Temp Key: X25519_MLKEM768, ...   ← hybrid KEM negotiated
Verify return code: 0 (ok)              ← ECDSA P-256 cert chain valid
```

### Step 8 — Switch KEM group to test a different protocol class

To isolate each key distribution protocol class, change `OQS_KEM_GROUP` in `docker-compose.yml` and restart Locust only (Nginx stays up):

```bash
# Test pure post-quantum key distribution
OQS_KEM_GROUP=mlkem768 docker compose up -d oqs-locust

# Test classical key distribution
OQS_KEM_GROUP=X25519 docker compose up -d oqs-locust

# Test higher-security hybrid
OQS_KEM_GROUP=P384_MLKEM1024 docker compose up -d oqs-locust
```

The Nginx log should then reflect the negotiated group for each run.

### Verification Quick Reference

| What to Check                    | Where to Look                                  | Confirms                               |
|----------------------------------|------------------------------------------------|----------------------------------------|
| `ssl_curve=X25519_MLKEM768`      | Nginx access log (`docker logs oqs-nginx`)     | **Hybrid** KEM negotiated              |
| `ssl_curve=P384_MLKEM1024`       | Nginx access log                               | **Hybrid** (high-sec) KEM negotiated   |
| `ssl_curve=mlkem768`             | Nginx access log                               | **Pure PQ** KEM negotiated             |
| `ssl_curve=X25519`               | Nginx access log                               | **Classical** KEM negotiated           |
| `ssl_curve=P-256`                | Nginx access log                               | **Classical** (broad-compat) KEM       |
| `ssl_protocol=TLSv1.3`           | Nginx access log                               | No TLS 1.2 downgrade                  |
| `Server Temp Key: X25519_MLKEM768` | `openssl s_client` output                    | Ephemeral hybrid KEM in handshake      |
| `Verify return code: 0 (ok)`     | `openssl s_client` output                      | ECDSA P-256 cert chain verified        |
| `OK ssl_curve=<group>`           | `curl https://localhost/health`                | Server-side group echoed correctly     |
| `0 failures` in Locust UI        | `http://localhost:8089`                        | All TLS handshakes succeeded at load   |
| `X-KD-Group` response header     | `curl -kv ...` headers                         | Per-request KEM group visible to client|

> ⚠️ **Troubleshooting:** If `ssl_curve=X25519` appears instead of an expected PQ group, the client did not advertise the OQS group. Check that `OPENSSL_GROUPS` is set in the Locust container (`docker exec oqs-locust env | grep OPENSSL`) and that the OQS OpenSSL build is active (`docker exec oqs-nginx openssl list -kem-algorithms | grep -i kem`).
